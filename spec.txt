一、 專案概述 (Project Overview)
------------------------------------------------------------------------
本專案名為「ChurchServe」，旨在開發一個直覺、溫暖且具質感的開源「教會小組服事表系統」。
系統核心目標是讓小組長（管理員）能輕鬆管理小組成員與產出每月服事輪值表，並允許
一般小組成員（免登入）在前端自由點選、登記自己可服事的日期與項目。

此專案採「開源（Open-Source）」架構設計，為了讓全球各教會能自由部署使用，系統必須
具備極高的擴充性、輕量化、低成本部署能力，並嚴格遵循開源安全隱私與資安防範規範。

------------------------------------------------------------------------
二、 核心設計原則與風格指南 (Design Principles & Style Guide)
------------------------------------------------------------------------
1. 風格調性：
   - 質感、溫暖、大地色系、具備教會與小組的凝聚感。
   - 避免冷冰冰的純技術或企業級管理後台感，改用柔和的配色（如米白、溫暖木質調、
     大地綠或溫潤暖橘作為點綴色）。
2. 使用者體驗：
   - 前端免登入操作必須極度簡化，確保年長者或不熟悉科技的組員也能輕鬆完成服事登記。
   - 支援響應式網頁設計 (RWD)，因多數組員會使用智慧型手機進行登記。

------------------------------------------------------------------------
三、 開源與資安規範 (Open-Source & Security Requirements) - 重要！
------------------------------------------------------------------------
為保護教會弟兄姊妹的個人隱私，並防止開源庫外洩敏感資訊，開發時必須強制執行以下規範：

1. 地理與隱私數據隔離（嚴禁上傳 Github）：
   - 小組真實成員名單、測試用之真實姓名、電話或聚會地點等隱私數據，【絕對禁止】寫死在程式碼中或作為初始化腳本（Seed Data）提交至 GitHub。
   - 系統必須提供 `.env.example` 範本，所有環境變數、密鑰與預設管理員資訊皆透過環境變數控制，並將 `.env` 加進 `.gitignore`。
   - 資料庫初始化時，僅能產生全虛構的範例姓名（例如：UserA, UserB）或留空，待部署後由各教會管理員於後台自行錄入。

2. 安全認證與防禦（防暴力破解）：
   - 管理員預設密碼：預設帳號 `admin` / 預設密碼 `admin123`。
   - 系統安全機制：
     * 初次部署成功並首次登入時，系統【強制】要求必須立即修改預設密碼，否則限制後台操作。
     * 後端登入 API 必須實作 Rate Limiting（頻率限制，如：同一 IP 1 分鐘內失敗超過 5 次鎖定 15 分鐘），防止自動化工具暴力破解預設密碼。
     * 密碼儲存必須使用強雜湊演算法（如 Argon2 或 bcrypt），嚴禁明文儲存。

3. 前端免登入行為的資安防禦：
   - 由於前端登記服事無需登入，為防範惡意爬蟲或惡意使用者透過 API 進行大量垃圾數據寫入（SQL Injection / XSS / DDoS），必須做到：
     * 欄位嚴格驗證 (Input Validation)：特別是「其他（自行填寫姓名）」與「備註」欄位，必須進行嚴格的 XSS 消毒過濾（Sanitization），防範惡意腳本植入。
     * API 請求頻率限制：對變更服事人員的 API 實作 Rate Limit，避免單一設備惡意洗板。
     * 樂觀鎖（Optimistic Locking）或防併發：處理同一格子被同時搶填的覆蓋問題。

------------------------------------------------------------------------
四、 Cloudflare 自動部署架構 (Cloudflare Deployment Architecture)
------------------------------------------------------------------------
本專案旨在達成「極致性價比」與「免伺服器運維」，全面支援自動部署至 Cloudflare 平台：

1. 前端與全端框架 (Cloudflare Pages)：
   - 支援整合 GitHub 儲存庫，當代碼推送到 main 分支時，透過 Cloudflare Pages CI/CD 自動建構並部署。
   - 建議採用支援 Edge Runtime 的全端框架（如 Next.js, Nuxt.js, Remix 或 SvelteKit）。

2. 後端與資料庫 API (Cloudflare Workers & D1/KV)：
   - 後端邏輯運行於 Cloudflare Workers (Edge Computing)，享受全球極低延遲與高併發防禦。
   - 資料庫解決方案：
     * 採用 Cloudflare D1 (Serverless SQL database)：儲存核心的關係型結構（小組、成員、服事日程表、分配明細）。
     * 搭配 Cloudflare KV（選用）：存放短期快取數據或靜態設定。
   - 所有 D1 綁定與環境變數，皆須透過 `wrangler.toml` 進行配置，並利用 Cloudflare Dashboard 的 Secrets 功能安全管理密鑰，不洩漏於開源代碼中。

五、 系統功能需求 (Functional Requirements)
------------------------------------------------------------------------

【功能模組 1：一般用戶端 - 服事表前端 (免登入)】
1. 小組切換與導覽：
   - 使用者進入首頁後，可透過下拉選單切換不同小組（初始預設且僅有「A24小組」）。
   - 可依「月份」切換檢視服事表（規劃從 2026 年開始的每個月）。
2. 服事表核心矩陣：
   - 自動生成該月份的「每個星期六」作為主要列。
   - 欄位架構：
     * 日期 / 星期 (主要為週六，但系統需保留管理員手動新增特殊日期如主日、節慶之彈性)
     * 破冰 (超好玩遊戲)
     * 敬拜讚美 (遇見神) - 司琴 / Youtube
     * 敬拜讚美 (遇見神) - 主領
     * 見證 (經歷神)
     * 信息分享
     * 報告
     * 備註 (供填寫特殊事項，如：某人不在、合併聚會等)
3. 自由登記機制：
   - 點擊特定日期的特定服事項目格子，會跳出選單供使用者選擇人員。
   - 人員清單來源：
     * 該小組內由管理員預設的「小組成員清單」。
     * 固定的特殊選項：「其他」（選取後，右側或下方需出現輸入框，供自由填寫姓名，此輸入框必須嚴防 XSS）。
   - 登記完成後即時更新，無需註冊或登入帳號。

【功能模組 2：管理員後台 (Admin Backend)】
1. 身份驗證：
   - 登入路徑：`/admin`，具備首入強改預設密碼與登入錯誤計數鎖定機制。
2. 小組管理 (Group Management)：
   - 具備新增、修改、刪除小組的功能。現階段預設建立「A24小組」。
3. 小組成員名單管理 (Member Management)：
   - 管理員可針對各別小組設定「可排進服事表的人員清單」。
   - 支援後台新增、編輯、刪除組員姓名，數據全數落實於 D1 資料庫中。
4. 特殊日程與備註控制：
   - 管理員可在後台針對特定月份手動新增「非週六的特殊日子」（如主日、節慶聚會）。
   - 管理員可進行「整行鎖定與合併」：設定特殊暫停事件（如：小組暫停一次或合併聚會），此時該整行欄位合併，不開放前端組員登記，僅顯示公告文字。
   - 管理員可直接編輯與覆寫任何一格服事人員或全局備註。

六、 資料庫資料模型設計建議 (Data Schema for Cloudflare D1)
------------------------------------------------------------------------
1. Groups (小組表)
   - id: integer PRIMARY KEY AUTOINCREMENT
   - name: text NOT NULL
   - created_at: text DEFAULT CURRENT_TIMESTAMP

2. Members (小組成員表)
   - id: integer PRIMARY KEY AUTOINCREMENT
   - group_id: integer REFERENCES Groups(id)
   - name: text NOT NULL
   - is_active: integer DEFAULT 1

3. ServiceItems (服事項目自定義表)
   - id: integer PRIMARY KEY AUTOINCREMENT
   - name: text NOT NULL
   - display_order: integer

4. DutySchedules (服事排班表)
   - id: integer PRIMARY KEY AUTOINCREMENT
   - group_id: integer REFERENCES Groups(id)
   - date: text NOT NULL
   - is_special_event: integer DEFAULT 0
   - event_title: text
   - remarks: text

5. DutyAssignments (服事分配明細表)
   - id: integer PRIMARY KEY AUTOINCREMENT
   - schedule_id: integer REFERENCES DutySchedules(id)
   - service_item_id: integer REFERENCES ServiceItems(id)
   - member_id: integer REFERENCES Members(id)
   - custom_member_name: text
